# 保险自动化项目 Mac 操作文档 v1.0.1

这份文档面向第一次接手项目的同事，目标是：

- 从 GitHub 下载项目
- 在本地安装运行环境
- 启动配置页面
- 填写本次执行表和总路线配置库
- 启动本地执行器
- 跑通后台导出、数据清洗、保险平台填写

注意：

- 本仓库公开版 **不包含真实账号密码**
- 首次运行时，后台管理系统如果有验证码，需要人工输入一次
- 如果投保流程配置为人工确认/人工支付，执行器会在对应步骤暂停，等待人工确认

## 2. 运行前准备

在 Mac 上需要先安装：

1. `Git`
2. `Node.js` 18 或更高版本
3. `npm`
4. Playwright 运行所需浏览器（项目里会自动安装 Chromium）

建议先在终端确认版本：

```bash
git --version
node -v
npm -v
```

如果 `node -v` 小于 `18`，请先升级 Node.js。

## 3. 下载项目

### 方式一：使用 Git 克隆

打开 Mac 终端，执行：

```bash
cd ~/Documents
git clone git@github.com:jz9287869-byte/insurance-automation.git
cd insurance-automation
```

### 方式二：从 GitHub 下载 ZIP

1. 打开 GitHub 仓库页面
2. 点击 `Code`
3. 点击 `Download ZIP`
4. 解压到本地，例如：

```text

```

然后在终端进入项目目录：

```bash
cd /Users/你的用户名/Documents/insurance-automation
```

## 6. 页面里有哪些区域

打开页面后，主要有 3 个区域：

### 6.1 本次执行表

这里填写“这一次要执行哪些路线”。

常用字段：

- 是否执行
- 路线名称
- 规格/套餐
- 出发日期
- 结束日期
- 确认投保方式
- 支付方式

说明：

- 每一行代表一条要执行的团期
- 只有勾选的行才会执行

### 6.2 总路线配置库

这里填写“每条路线默认使用什么保险规则”。

常用字段：

- 路线名称
- 路线类型
- 关键词
- 公司主体
- 分类
- 保司
- 产品
- 计划
- 提前天数
- 保险天数
- 备注格式

核心规则：

- `起保时间 = 出发日期 - 提前天数`
- 如果 `提前天数 = 0`，则起保时间就是出发日期
- `保险天数` 由这里控制，例如 `7`
- `止保时间` 不需要手动填，保险平台会根据起保时间和保险期限自动联动

### 6.3 平台账号

这里填写：

#### 后台管理系统

- 登录地址
- 账号
- 密码
- 二次密码/安全密码（如果有）

#### 保险平台

- 登录地址
- 账号
- 密码

说明：

- 这些内容保存在本机浏览器或本机 `automation/config.json`
- 不会自动上传到 GitHub

## 7. 保存和开始执行的区别

### 7.1 保存

点击页面右上角 `保存`：

- 只会保存到当前浏览器本地
- 不会启动自动化
- 不会写入执行器状态

适合：

- 先维护配置
- 暂时不执行

### 7.2 开始执行

点击页面右上角 `开始执行`：

- 会把当前配置提交给本地执行器
- 会写入本地 `automation/config.json`
- 会真正启动自动化流程

前提是：

- 你已经先执行了 `npm run dashboard`
- 页面是通过 `http://localhost:17820` 打开的

## 9. 自动化执行时会发生什么

正常情况下，执行器会按这个顺序工作：

1. 打开后台管理系统
2. 如果未登录，则输入账号密码
3. 如果后台有验证码，则等待人工输入验证码并点击登录
4. 进入订单列表
5. 状态多选：
   - `已付款`
   - `已付尾款`
6. 按路线、规格、出行时间筛选
7. 切换 `100条/页`
8. 勾选全选
9. 导出订单列表
10. 进入导出列表，轮询刷新并下载订单列表文件
11. 进入销转表
12. 按开始时间、路线名称、套餐筛选
13. 导出销转表
14. 进入导出列表，轮询刷新并下载销转表文件
15. 清洗订单列表和销转表
16. 解析队长安排，把队长并入旅客
17. 打开保险平台
18. 选择：
   - 分类
   - 保司
   - 产品
   - 计划
   - 保险期限
   - 起保时间
19. 粘贴被保人名单
20. 核对投保人公司主体
21. 处理投保材料弹窗
22. 根据配置决定：
   - 自动确认投保 / 人工确认
   - 自动支付 / 人工支付

## 10. 队长并入旅客说明

项目目前支持从销转表的 `队长安排` 中解析主队信息。

例如：

```text
主队：庞宇婷（兔子） 身份证：440804199902191347
```

会自动提取：

- 姓名：`庞宇婷`
- 证件号码：`440804199902191347`
- 出生日期：`1999-02-19`
- 性别：根据身份证自动推算

并把队长并入最终要投保的旅客名单中。

说明：

- 只取括号外的真实姓名
- 不取括号里的昵称

## 11. 证件类型说明

当前脚本会保留旅客的：

- 证件类型
- 证件号码

如果是身份证，会自动推算：

- 性别
- 出生日期

如果后续有护照、港澳通行证或其他证件，需要继续按保险平台实际支持格式补强。

## 12. 命令行直接运行方式

除了网页点击“开始执行”，也可以直接用命令行跑。

### 12.1 只做配置预检

```bash
npm run dry-run -- --config automation/config.json
```

### 12.2 正常执行

```bash
node automation/run.mjs --config automation/config.json
```

### 12.3 让脚本自动去后台导出

```bash
node automation/run.mjs --config automation/config.json --export true
```

### 12.4 只验证保险平台，不重新走后台导出

```bash
node automation/run.mjs \
  --config automation/config.json \
  --summary automation/outputs/summary.json \
  --skip-admin true
```

适合场景：

- 后台导出已经完成
- 只想重新验证保险平台字段选择是否正确

### 14.1 看终端日志

- 正在登录后台



## 15. 常见问题

### Q1：为什么点击“开始执行”提示未连接本地执行器？

原因：

- 没有先运行 `npm run dashboard`
- 或者你打开的是 `file:///.../index.html`

解决：

```bash
cd /Users/你的用户名/Documents/insurance-automation
npm run dashboard
```

然后访问：

```text
http://localhost:17820
```

### Q2：为什么 `npm run install-browsers` 报找不到脚本？

原因：

- 你不在项目目录里执行命令

先进入项目目录：

```bash
cd /Users/你的用户名/Documents/insurance-automation
```

再执行：

```bash
npm run install-browsers
```

### Q3：后台验证码怎么办？

首次登录时，如果后台弹出验证码：

- 人工输入验证码
- 人工点击登录

脚本会等待登录成功后继续执行。

### Q4：止保时间要不要手动填？

不要。

规则是：

- 脚本只设置 `起保时间`
- 脚本只设置 `保险期限`
- `止保时间` 由保险平台联动自动生成

### Q5：起保时间怎么计算？

规则：

```text
起保时间 = 出发日期 - 提前天数
```

例如：

- 出发日期：`2026-05-27`
- 提前天数：`0`

则：

- 起保时间：`2026-05-27`

如果提前天数是 `2`，则起保时间会变成：

- `2026-05-25`
